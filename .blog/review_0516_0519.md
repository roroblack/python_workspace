## ⚠️ GAP — 미반영 또는 부족한 항목 (8개)

### GAP-1 : `data/repository.py` 계층 설명 없음 (심각도: 중)
- **현재 상태** : CH01 역할 표에 "Repository 패턴" 단어만 등장
- **실제 코드** : `data/service.py` 는 `CrawlService(crawler, repository=None)` 로 repository 주입. `CrawlService` → `Repository` → `db.py` 3계층 구조가 아키텍처의 핵심
- **개선 방향** : CH02 또는 별도 섹션에서 3계층 구조 다이어그램/코드 추가

### GAP-2 : `crawling/models.py` 데이터 모델 코드 없음 (심각도: 중)
- **현재 상태** : `CarRegistrationItem`, `FaqItem`, `StationItem` 언급 없음
- **실제 코드** :
  ```python
  @dataclass
  class CarRegistrationItem: stat_year:int; region:str; count:int
  class FaqItem: source_site:str; category:str; question:str; answer:str
  class StationItem: station_name:str; address:str|None; lat:float|None; ...
  ```
- **개선 방향** : CH03 크롤러 섹션 앞에 "크롤러가 반환하는 타입 계약" 으로 models.py 코드 블록 추가

### GAP-3 : `HyundaiFaqCrawler` 코드/설명 없음 (심각도: 소)
- **현재 상태** : CH03에서 `EvFaqCrawler`의 봇 챌린지 우회 코드만 상세히 기술. hyundai.com 크롤러는 테이블 출처 셀에만 언급
- **실제 코드** : `HyundaiFaqCrawler` 는 별도 클래스, 대상 URL = `https://www.hyundai.com/kr/ko/e/customer/guide/purchase/estimate-fcev`
- **개선 방향** : "FAQ 크롤러 2개 비교" 형태로 hyundai.com 크롤러 간략 코드 추가

### GAP-4 : `crawler_station.py` 의 CSV 다운로드 플로우 코드 없음 (심각도: 소)
- **현재 상태** : CH03 테이블에 "Playwright → CSV 다운로드" 언급만 있고, 실제 코드는 CH05 asyncio 충돌 해결 코드에서 `_get_csv`/`_run_playwright` 만 나옴
- **실제 코드** : `CANONICAL_CSV_NAME = "hydrogen_stations.csv"`, `_async_download()` → `tmp_down/` 저장 → pandas 읽기 플로우
- **개선 방향** : CH03에 `crawler_station.py` 의 다운로드-파싱 흐름 코드 블록 추가

### GAP-5 : Python 버전 미언급 (심각도: 소)
- **현재 상태** : CH06 스택 표에 Python 행 없음
- **실제 환경** : Python 3.12.7
- **개선 방향** : CH06 스택 표 첫 행에 Python 3.12.7 추가

### GAP-6 : `faq` 테이블에 `source_site` 컬럼 부재 — 불일치 미설명 (심각도: 소)
- **현재 상태** : `FaqItem.source_site` ('ev.or.kr' | 'hyundai.com') 는 크롤러 설명에 나오지만, DDL의 `faq` 테이블 (`faq_id, question, answer` 3컬럼) 에 `source_site` 가 없음
- **불일치 의미** : 출처 구분은 런타임 모델에만 있고 DB에는 저장 안 됨 (의도인지 누락인지 포스트가 침묵)
- **개선 방향** : CH02 또는 CH05 시행착오에 "FAQ 출처 구분을 DB에 저장하지 않은 이유/현황" 한 줄 추가

### GAP-7 : CH03 → CH04 챕터 연결 bridge 없음 (심각도: 소)
- **현재 상태** : CH01에는 `<div class="bridge">` 가 있지만 CH03 끝에 없음. CH04가 갑자기 시작됨
- **개선 방향** : CH03 끝에 "크롤러가 데이터를 가져왔다. 그 데이터를 어디서 보여주는가 — 단일 app.py 의 구조." bridge 추가

### GAP-8 : `tmp_down/` 임시 저장 경로 미언급 (심각도: 미미)
- **현재 상태** : Excel/CSV 다운로드 흐름에서 파일이 어디에 저장되는지 언급 없음
- **실제 코드** : `crawler_molit.py` 는 `tmp_down/` 폴더에 xlsx 저장 후 파싱
- **개선 방향** : CH03 crawler_molit 코드 블록에 주석으로 `tmp_down/` 경로 언급
